# 📅 Week 01 - Day 01
# DevOps Fundamentals

---

# 📖 What is DevOps?

> **DevOps is a culture, a set of practices, and a collection of tools that help development and operations teams work together to build, deliver, and maintain software faster, more reliably, and with higher quality.**

---

# 🎯 Why was DevOps Created?

DevOps was created to remove the gap between Development and Operations by encouraging collaboration, reducing manual handoffs, and enabling faster and more reliable software delivery through shared responsibility and automation.

---

# ❓ Why Isn't DevOps Just Tools?

DevOps is **not** a tool.

It is a **culture** and a **set of practices**. Tools such as Git, Docker, Kubernetes, and GitHub Actions help implement DevOps practices, but they are not DevOps itself.

---

# 💡 Explain DevOps in Simple Words

Developers and Operations work together throughout the software lifecycle to deliver software faster, more reliably, and with continuous improvement.

> **DevOps did not invent software development. It changed how software development is delivered.**

---

# 🏃 Traditional Approach vs DevOps

## Before DevOps

Development and Operations worked like a relay race.

```
Developers
      ↓
Operations
      ↓
Issues
      ↓
Developers
      ↓
Repeat
```

Each team waited for the previous team to finish before starting their own work.

---

## With DevOps

Teams collaborate from the beginning.

```
Operations
        │
        │ Discuss deployment requirements
        ▼
Developers
        │
        │ Build application + Docker container
        ▼
Operations
        │
        │ Deploy container
        ▼
Users
```

### Result

- Better communication
- Shared responsibility
- Less waiting
- More automation
- Faster software delivery

# 🚀 Why Companies Use DevOps?

| Reason | Benefit |
|---------|----------|
| Faster Delivery | Users receive new features quickly |
| Better Collaboration | Development and Operations work together |
| Higher Quality | Fewer bugs and more reliable releases |
| Faster Bug Fixes | Problems are resolved quickly |
| Automation | Reduce repetitive manual tasks |
| Reliable Deployments | Consistent and repeatable deployments |
| Continuous Improvement | Software evolves through continuous feedback |

---

## If a Company Releases Software Every 6 Months...

Possible problems:

- Bugs remain unfixed for long periods.
- Customers lose interest.
- Competitors release features faster.
- Customer trust decreases.
- Security vulnerabilities remain exposed.

---

# ⚙️ Examples of Automation

Tasks that can be automated:

- Build the application
- Run tests
- Create Docker images
- Deploy to Azure
- Restart the application
- Notify when deployment fails

# 🔄 SDLC vs DevOps

| Traditional SDLC | DevOps |
|------------------|---------|
| Sequential process | Continuous process |
| Teams work separately | Teams collaborate |
| Manual deployment | Automated deployment |
| Feedback comes late | Continuous feedback |
| Large releases | Small, frequent releases |
| Testing mostly at the end | Continuous testing |
| Slow updates | Faster incremental updates |
| Focus on completing a project | Focus on continuously improving a product |

---

## Key Takeaway

> **SDLC tells us _what_ stages are needed to build software. DevOps improves _how_ those stages are carried out through collaboration, automation, continuous feedback, and continuous improvement.**

---

## Did DevOps Replace SDLC?

No.

DevOps enhances the Software Development Life Cycle by improving collaboration, automation, continuous feedback, and continuous delivery throughout the lifecycle.

---

## Biggest Difference

Traditional SDLC focuses on teams working sequentially.

DevOps encourages all teams to collaborate continuously throughout the entire software lifecycle rather than waiting for their individual phase.

# 🌱 Agile vs DevOps

| Agile | DevOps |
|--------|---------|
| Software development methodology | Culture & practices |
| Focus on development | Focus on development, deployment & operations |
| Sprint-based development | Continuous delivery & operations |
| Customer feedback | System monitoring + customer feedback |
| Developers | Developers + QA + Operations |
| **Build the right thing** | **Deliver it the right way** |

# 🔄 DevOps Lifecycle

| Stage | SOS Project Example |
|--------|---------------------|
| Plan | Competitor analysis, requirements, wireframes |
| Code | React + Spring Boot development |
| Build | Generate production build & JAR |
| Test | Test login, contacts and SOS workflow |
| Release | Approve MVP v1.0 |
| Deploy | Host on Azure |
| Operate | Keep the application running |
| Monitor | Observe performance and errors |
| Feedback | Gather user feedback for improvements |

# 🤝 DevOps Culture & Core Principles

| Principle | Description |
|------------|-------------|
| 🤝 Collaboration | Developers, QA and Operations share responsibility |
| ⚙️ Automation | Reduce repetitive manual tasks |
| 🔄 Continuous Integration (CI) | Frequently integrate and validate code changes |
| 🚀 Continuous Delivery / Deployment | Keep software ready for release or deploy automatically |
| 📈 Monitoring & Feedback | Monitor the system, collect feedback and continuously improve |
